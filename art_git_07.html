Git: Commit using different accounts

Posted by: Huehueocelotl
Home

The problem

Suppose that you have different Git-Hub accounts, say, GitUser01 and GitUser02. The local repository LocalRepoU1 is created in the first account and the LocalRepoU2 is created in the second.

Problem
You make a change in LocalRepoU2, but in the history of commits, it says that the change was made by GitUser01.
Expected behaviour
Any change in LocalRepoU1 should be made by GitUser01 and any change in LocalRepoU2 should be made by GitUser02.

What is going on?

The global configuration in you machine uses the first user you registered, say GitUser01.

To check your global configuration put 

git config --global --list

user.name=GitUser01
user.email=user.01@foomail.com    
core.editor=emacs

How to fix it?

To check the user in the local repository ~/Gits/LocalRepoU2.io, in the terminal you type 

git config user.name ; git config user.email

GitUser01
user.01@foomail.com

To change the user and email for the local repository ~/Gits/LocalRepoU2.io, in the terminal you type 

git config user.name "GitUser02"
git config user.email "user.02@foomail2.com"

Check all is well configured before making the next commit: 

git config user.name ; git config user.email

GitUser02
user.02@foomail2.com

Check the config file.

Finally, notice that every local repository has a hidden folder ./git, there you can find the config, wich contains some more info. 

cat .git/config
