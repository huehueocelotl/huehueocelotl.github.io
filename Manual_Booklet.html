Manual para instalar linux

Objetivo

Instalar un sistema operativo de tipo Linux/Debian en tu computadora. Con este sistema operativo tendrás acceso a diversas aplicaciones y programas que serán de gran utilidad para la programación en diversos lenguajes.

Antecedentes

Se le llama sistema operativo a un conjunto de programas sencillos y otras utilidades con el objetivo de hacer de una computadora una herramienta funcional. Existen varios sistemas operativos, entre los más usados se encuentran:

  • Windows
  • MacOS
  • Linux

La familia de sistemas operativos del tipo Linux consiste de varias distribuciones. Entre ellas hay muchas similitudes y algunas diferencias sutiles y otras no tanto.

Para esta práctica hemos elegido la distribución Debian de Linux ya que es muy conocida por su estabilidad y su gran documentación en la red.

Otra de las ventajas que tienen los sistemas Linux es que son sistemas libres (de pago, de uso, de distribución y de modificación), por lo que no será necesario pagar ningún tipo de licencia para su uso.

Alguna de las desventajas de usar esta distribución es que los programas tardan un poco más en poder actualizarse a las versiones más recientes.

Puedes encontrar más información sobre Linux/Debian en los siguentes enlaces:

  • https://www.debian.org/
  • https://wiki.debian.org/es/DebianIntroduction

Práctica

Instalar de manera exitosa el sitema operativo Linux/Debian

Necesitarás de una memoria USB que puedas utilizar y borrar completamente.

Procedimiento:

Crea un instalador con una memoria USB

Desde una computadora con sistema operativo Windows, puedes ayudarte de los siguentes pasos:

  1. Ingresa a https://www.debian.org/download
  2. Haz click en "download the .iso file" /debian-12.1.0-amd64-netinst.iso/.
  3. Si estás usando una computadora con Windows, descarga el programa Win32 Disk Imager.
  4. Usando Win32 Disk Imager, crea un instalador en la memoria USB con el archivo /debian-12.1.0-amd64-netinst.iso/.

Iniciar el instalador desde el arranque de la computadora

Para iniciar la instalación ahora debes

  1. Apagar tu computadora.
  2. Insertar la memoria USB.
  3. Encender la coputadora.

Si tu instalador USB fue creado exitosamente, verás el menú de instalación del sistema.

¿Qué hacer si no pasa nada diferente?

En este caso se tiene que forzar a la computadora el acceso al BIOS. Generalmente se puede acceder al BIOS presionando alguna de las teclas F2 o F11 justo al encender tu computdora. Una vez dentro del BIOS, habilita la opción de arranque desde USB.

Proceder con la instalación de tu nuevo sistema operativo

Una vez que tienes el menú de instalación, sigue paso a paso las instrucciones. Toma en cuenta lo siguiente:

  • Asegura de tener respaldada la información de manera externa, pues es probable que tengas que formatear todo el disco duro de tu computadora.
  • Asegura que tienes una buena conexión a internet.
  • Si no tienes experiencia con particiones, usa la recomendada por defecto.
  • Recuerda de manera particular tu nombre y contraseña de administrador (super user's name & password).
  • Existen diferentes entornos de escritorios para Linux/Debian. Nosotros estaremos mostrando principalmente Gnome.

Conclusiones

¡Felicidades!

Ahora tines un sistema operativo libre funcionando en tu computadora.

Conociendo Linux/Debian

Objetivo

Conocer algunos programas para trabajar en Linux/Debian.

Antecedentes

Una vez que tienes un nuevo sistema operativo con un entorno de escritorio gráfico, puedes abrir algunos programas que te ayudarán a administrar tu computadora.

En esta práctica nos concentraremos en tres tareas muy importantes:

  • El explorador de archivos y carpetas
  • Instalador gráfico de software
  • Un editor de texto

Práctica

Explorador de archivos de Gnome

Si instalaste el entordno de escritorio GNOME, Files es el nombre del explorador de archivos y carpetas por predeterminado aunque también es conocido como Nautilus. Este programa te permitirá accesar y organizar tus archivos y su uso es muy intuitivo.

Crear una carpeta de trabajo

Realiza los siguientes pasos:

  1. Localiza la carpeta home.
  2. Localiza la carpeta Documents.
  3. Dentro de la carpeta Documents, crea una nueva carpeta llamada Practicas.

Instalar un nuevo programa (usando software)

El programa Software te permite encontrar e instalar/desinstalar programas en Linux/Debian. Lo hace de manera segura e intuitiva presentando iconos grandes y descripciones útiles de las distintas apps o los paquetes a instalar.

Instala el programa Notes para GNOME.

  • Abre el programa Software.
  • Busca la aplicación Notes a instalar
  • Introduce tu constraseña de administrador y procede con la instalación.

Mi primer editor de texto para programar

Le herramienta principal para un programador es sin lugar a dudas un editor de texto plano y sencillo. Recuerda que nuesto objetivo será escribir programas y no obras literarias, por lo que nuestro editor de texto no debe enfocarse en cambiar la fuente y estilo de un párrafo.

Al contrario, necesitamos de una herramienta que nos permita escribir instrucciones de manera clara y sencilla. Para este fin hay una infinidad de opciones, hay muchos editores de texto para propósitos específicos y existen otros editores de texto que se pueden usar para escribir todo tipo de programas en diferentes lenguajes.

  • gedit es el editor de texto mas sencillo de usar dentro del entorno GNOME. (Fácil)
  • eclipse es un editor popular dentro de la categoria de IDEs (integrated development environment) y está disponible para usar en Linux. (Medio)
  • vim es un editor gratis y de libre uso, altamente configurable y disponible en casi cualquier sistema tipo UNIX. (Difícil)
  • emacs es un editor altamente configurable, free/libre y junto con vim, usada y preferida por varias generaciones de programadores. (Difícil)

En esta práctica usaremos gedit, pero te invitamos a que no te pierdas la oportunidad de aprender al menos uno de los editores entre emacs y vim.

Crear un archivo de texto

  • Abre el editor gedit.
  • Introduce el texto: "Este es mi primer archivo de texto en Linux/Debian".
  • Guarda el archivo dentro de la carpeta Practicas con el nombre mi_texto.txt.

Conlusiones

Este ha sido solo una primera vista a tu nuevo sistema operativo Linux/Debian. En adelante estaremos escribiendo programas dentro de Linux/Debian y es importante llevar un orden y ciertas reglas para escribir los archivos. Las siguientes son recomendaciones que a lo largo de la experiencia hemos encontrado útiles y no son para nada reglas establecidas, pero han evitado muchos dolores de cabeza

  • El nombre de un archivo o carpeta nunca inicia con un número.
  • El nombre de un archivo o carpeta nunca lleva caracteres especiales ni acentos.
  • El nombre de un archivo o carpeta no lleva espacios en blanco, es mejor usar el guión bajo: _.

Es importante tomar en cuenta que el sistema Linux/Debian es sensible a mayúsculas (case sensitive) es decir, el nombre del archivo File es distinto del archivo file.

Conocer los programas disponibles para Linux/Debian que permitirán realizar diversas actividades llamadas de oficina.

Es posible que podamos realizar estas tareas con programas que ya conocemos, por ejemplo, para Linux se ecuentran disponibles navegadores como Firefox. Por otro lado, no todos los programas son compatibles para Linux. Ésto a propiciado que la comunidad de programadores de software libre desarrolle programas alternativos a los programas propietarios estándar.

Conclusiones

Es verdad, pasar de un sistema operativo a otro no es cambio tan sencillo, especialmente si estás acostumbrado a ciertos programas que no se encuentran disponibles en Linux por ejemplo.

Sin embargo la noticia no es tan mala. Linux se ha desarrollado muchísimo todo este tiempo, y los programas de oficina compatibles con Linux son verdaderamente competitivos. Es importante también aclarar que, el verdadero poder de un sistema operativo Linux tiene que ver mucho más con la programación, administración de servidores y mantenimiento de bases de datos.

Programas como grep, awk, vim, emacs, crontab, entre muchos otros dentro de la familia de los llamados tipo UNIX, son muy poderosos para la automatización de procesos, inclusive para muchas tareas de oficina.

Te invitamos a seguir conociendo estas y más herramientas.

Conociendo la Terminal

Objetivo

Conocer y usar algunos comandos básicos de la terminal de linux.

Antecedentes

La terminal es la herramienta que caracteriza a todos los sistemas operativos de tipo UNIX, entre los que se cuentan los sistemas Linux. Tiene múltiples nombres: Terminal/Shell/Consola/Línea de comandos. Para hacerlo aún más confuso, existen varios tipos de Terminales: Bash/Z shell/Korn shell/C shell/Zsh. Cada una de estas terminales tienen características específicas y no son totalmente compatible una con otra. En Linux/Debian, la terminal predeterminada es GNU Bash (the Bourne Again SHell). En el entorno GNOME el programa Terminal es un emulador que da accesso al entorno Bash que se usa para usar y correr programas disponibles en tu sistema.

Práctica

A continuación daremos nuestros primeros pasos usando la terminal. Cabe resaltar que la terminal es una herramienta de una inmensa utilidad y tomará mucho tiempo para poder usarla efectiva y eficientemente. Sin embargo, un comienzo es todo lo que necesitamos para poco a poco ir dominando esta poderosa herramienta que tenemos frente a nosotros. Con ella, tenemos acceso a los programas de nuesta computadora, y podemos dar instrucciones directas a nuestro sistema para realizar ciertas acciones. En esta práctica nos concentraremos en comandos para movernos dentro del sistema de archivos de nuestro ordenador.

¡Abre tu terminal y pongamos en acción los siguentes comandos!

Archivos, ejecutables y permisos

Objetivo

Conocer la differencia entre un archivo de texto, un programa y un ejecutable. Leer de manera detallada la salida de el comando ls -l Conocer los tipos de permisos en Linux.

Antecedentes

De los tipos de archivos más básicos se encuentran los archivos de texto. En ellos podemos ingresar información para que otro ser humano pueda leer su contenido. Pero también es posible ingresar texto con instrucciones para nuestra computadora, generalmetne en un lenguaje en específico, para lo cual necesitaremos o de un intérprete, es decir, un programa que ejecute de manera secuencial las instrucciones escritas en el archivo, o es posible que necesitemos de un compilador, el cual se encargará de crear un archivo el cual ejecuta todas las instrucciones que se especificaron. Un folder es también un archivo de texto el cual está apuntando a cierta parte reservada de la memoria.

Es decir, debemos saber distinguir entre folders, archivos de texto y archivos ejecutables. Cada uno de estos tipos de archivos y directorios tienes asignado derechos de acceso para tres conjuntos de usuarios:

  1. Propiatario (owner)
  2. Miembros del grupo propietario del archivo
  3. Para cualquier usuario

En esta práctica explicaremos con mayor detalle el uso de los comandos ls y chmod.

Práctica

Chmod para archivos

Para esta práctica nos ubicaremos en la carpeta Practicas en donde hemos creado el archivo de texto mi_texto.txt 

cd /home/username/Documents/Practicas/
ls -l 

-rw-r--r--  1  username  username  51  Sep  17  19:18  mi_texto.txt

A continuación explicamos ¿Cómo leer "-rw-r--r--" ?

De izquierda a derecha tenemos que:

  • El primer caracter "-" indica el tipo de archivo, en este caso indica que se trata de un arvhivo regular. Cuando se trata de un directorio, el primer caracter que aparece es la letra "d".
  • Los siguientes tres caracteres: "rw-" nos indica los permisos para el propietario del archivo, la letra "r" indica permiso de lectura (read), la letra "w" indica el permiso de escritura (write), y la última letra "x" indica el permiso de ejecución y que en este caso no está autorizado "-".
  • Los siguientes tres caracteres: "rw-" nos indica los permisos para el grupo de usuarios propietarios del archivo, entonces la terna "r–" debe leerse como que el grupo de usuarios si tiene permisos de lectura pero no tiene permiso ni de escritura ni de ejecución.
  • Los últimos tres caracteres nos indica los permisos para todos los usuarios, en este caso la terna "r–" indica que sólo se tienen permisos de lectura y/o ejecución.

Conversión a binario

Puedes completar la siguente lista para cubir con todas las opciones

Literal Binario Decimal Descripción
rwx 111 7 Sin restricciones
rw- 110 6 Permiso para leer y escribir, pero no ejecutar
r-x 101 5 Permiso para leer y ejecutar pero no para escribir
r-- 100 4 Sólo con permiso para leer

Por ejemplo si escribimos en la terminal 

cd /home/username/Documents/Practicas/
chmod 642 mi_texto.txt
ls -l

: total 4
: -rw-r---w- 1 username username 51 Sep 17 19:18 mi_texto.txt

En este caso debemos leer el permiso 642 de la siguente manera:

  • 6 indica los permisos rw- para el propietario del archivo.
  • 4 indica los permisos r-- para miembros del grupo propietario del archivo.
  • 2 indical los permisos -w- para todos los demás usuarios.

Describe la siguiente salida 

chmod 644 mi_texto.txt
ls -l

: total 4
: -rw-r--r-- 1 username username 51 Sep 17 19:18 mi_texto.txt

Chmod para carpetas

El comando chmod puede usarse en carpetas y tenemos los siguientes atributos

  • r - permite que el contenido de la carpeta puesa ser enlistado (si el permiso x está activado).
  • w - permite que el dentro de la carpeta se puedan crear, eliminar o renombrar archivos (si el permiso x está activado).
  • x - Permite el acceso a la carpeta, por ejemplo al ejecutar cd dir.

Con esta información, realiza las siguentes intrcucciones dentro de la carpeta Practicas.

  1. Crea un nevo directiorio en la carpeta temp_dir
  2. Cambia los permisos con chmod a 666
  3. Intenta ingresar a la carpeta cd temp_dir
  4. Usa ls -l para entender el mensaje de error
  5. Cambia los permisos con chmod a 755
mkdir temp_dir

ls -l /home/username/Documents/Practicas/

total       8                                                     
-rw-r--r--  1  username  username    51  Sep  17  19:18  mi_texto.txt 
drwxr-xr-x  2  username  username  4096  Sep  24  19:17  temp_dir     

chmod 666 temp_dir
cd temp_dir

bash: cd: temp_dir/: Permission denied

drwxr-xr-x  2  username  username  4096  Sep  24  19:17  temp_dir

Crear un archivo ejecutable

Hemos hablado de archivos con permiso de ejecución, éstos no son otra cosa mas que programas con instrucciones para bash. Por ejemplo vamos a crear el archivo mi_prgrama.prog y lo haremos desde la terminal.

  • Primero aprenderemos un poco del programa echo
echo "este es un texto"
  • Ahora usaremos el símbolo > para re-direccionar la salida hacia un archivo de texto
echo "ls -l" > mi_programa.prog
  • Cambia los permisos del archivo para que todos los usuarios puedan ejecutarlo como un programa
chmod 777 mi_programa.prog
  • Para ejecutar el programa simplemente escribe ./mi_programa.prog
./mi_programa.prog

Observa que el resultado será el mismo que si usas el comando ls -l, pues esa es la instrucción dentro del archivo.

  • Borra el archivo
rm mi_programa.prog

Más adelante haremos muchos más programas tanto para la terminal como en otro tipo de lenguajes.

Conclusiones

Terminaremos esta sección con las siguentes sugerencias. Usualmente los Folders de archivos llevan los permisos 755. Por lo tanto, para establecer el permiso 755 a todos los folders dentro de la carpeta Practicas hacemos 

find /home/username/Practicas -type d -exec chmod 755 {} \;

Por otro lado, para cambiar el permiso de todos los archivos dentro de una carpeta, escribimos 

find /opt/lampp/htdocs -type f -exec chmod 644 {} \;

Y nunca está de mas echarle un vistazo al manual 

man chmod

Recuerda siempre tener cuidado con el comando rm -r, para remover/eliminar archivos y folders.

Configurarndo APT

Objetivo

Configurar y utilizar apt para hacer actualizaciones de paquetes.

Antecedentes

Como ya lo deberías ir sospechando, no todos los programas para Windows corren en Linux/Debian. Sin embargo hay programas como el navegafor firefox que se encuentra disponible en la mayoria de los sistemas operativos.

La filosifía es la siguente: la colección de programas que puedes instalar en Linux/Debian, se encuentran en los llamados repositorios oficiales de Debian. Esto tiene muchas ventajas pues el software disponible es seguro. La desventaja es que debes mantener actualizada tu configuración a los repositorios.

La configuración se encuentra en un archivo llamado souces.list, que sólo puede ser modificado por el administrador del sistema (es decir el super user que configuraste durante la instalación).

Práctica

Para ésta práctica, ingresaremos como super usuario: 

su
Password:

Si ha sido exitosa la operación, el símbolo $ cambiará por #.

Localizar el archivo sources.list

(Como administrador) Ingresar al directorio /etc/apt/: 

cd /etc/apt/
ls -l

Si el archivo no existe, no te preocupes. Puedes crearlo siguiendo los mismos pasos.

Abre el archivo sources.list

Para abrir el archivo (o crearlo en caso de que no exista) usando gedit como super usuario (administrador), simplemente escribimos 

gedit sources.list

Esto abrirá un archivo de texto.

Edita el archivo

¿Qué debe decir el archivo?

El siguente es un ejemplo del contenido del archivo sources.list para Linux/Debian 12/Bookworm (stable). 

deb http://deb.debian.org/debian bookworm main
deb-src http://deb.debian.org/debian bookworm main

deb http://deb.debian.org/debian-security/ bookworm-security main
deb-src http://deb.debian.org/debian-security/ bookworm-security main

deb http://deb.debian.org/debian bookworm-updates main
deb-src http://deb.debian.org/debian bookworm-updates main

Si te preguntas, ¿cómo saber qué version de Linux/Debian tengo instalado? Puedes ingresar el siguente comando 

lsb_release -a

Distributor ID:	Debian
Description:	Debian GNU/Linux 12 (bookworm)
Release:	12
Codename:	bookworm

Actualizar

Ahora estas listo para correr dos de las instrucciones más comunes para mantener tus paquetes actualizados con las nuevas mejoras:

Simplemente corre los siguientes comandos como super usuario (administrador) 

apt update
apt upgrade

Una vez que hayas editado correctamente el archivo, guarda los cambios desde gedit. Para salir del modo super user (o administrador) simplemente escribe en la terminal 

exit

Notarás que el prompt regresa al símbolo $.

Más información

En el siguente enlace podemos encontrár más información sobre la configuración del arhivo sources.list.

https://wiki.debian.org/SourcesList

Entre la información relevante encontramos

main cosiste de paquetes del tipo DFSG-compliant. Estos son paquetes considerados paquetes Debian propiamente.
contrib contiene paqueted del tipoi DFSG-compliant, pero con dependencias posiblemente fuera de main.
non-free contiene software que no satisface las condiciones para ser de tipo DFSG.

Conclusiones

Este fue solo una probadita de apt. Sin lugar a duda usarás este comando muy a menudo. La documentación es muy completa y ahroa puedes darte un paseo por las opciones más comunos de apt:

  • apt -list list packages based on package names/enlista los paquetes en base a su nombre
  • apt -search search in package descriptions/busca las descripciones de un paquete
  • apt -show show package details/muestra los detalles de un paquete
  • apt -install install packages/instala paquetes
  • apt -reinstall reinstall packages/reinstala paquetes
  • apt -remove remove packages/elimina paquetes
  • apt -autoremove automatically remove all unused packages/automaticamente elimina todos los paquetes que no se usan
  • apt -update update list of available packages/actualiza la lista de paquetes disponibles
  • apt -upgrade upgrade the system by installing/upgrading packages/actualiza el sistema instalando nuevas versiones
  • apt -full-upgrade upgrade the system by removing/installing/upgrading packages / actualiza el sistema operativo
  • apt -edit-sources edit the source information file / edita el archivo de información de fuentes
  • apt -satisfy satisfy dependency strings / satisfacer dependencias de un paquete

Este programa es uno de los más antiguos en el mundo de la programación y tiene muchísimas aplicaciones en la acutalidad. El programa grep consta de reglas de busqueda de texto sencillas, con las cuales pueden construirse nuevas reglas de búsqueda increíblemente complejas. Primer programa en C

Objetivo

Tener los primeros ejemplos funcionales de un código escrito en el lenguaje C. Escribir las instrucciones de compilación en un Makefile y correr el programa compilado desde la Terminal

Antecedentes

Este es uno de los lenguajes de programación más populares desde su aparición en 1972. Muchas de las aplicaciones en sistemas UNIX han sido desarrollados en C. Entre sus puntos fuertes se encuentra la eficiencia del código.

La referencia clásica para aprender a programar en este lenguale es el libro llamado The C programming Language en su segunda edición y escrito por Brian W. Kernighan y Dennis M. Ritchie (usualmente aberviado como K&R)

C es un lenguaje de programación que ayuda a entender varios conceptos fundamentales de programación y al mismo tiempo es uno de los lenguajes más difíciles de aprender.

En este tutorial usaremos los siguientes elementos:

  • Librería #include <sdio.h>. En C existen varias librerías las cuales conforman la llamada librería estándard de C. Entre estas se encuentra la librería stdio.h. Cada librería contiene a su vez una colección de funciones pre-determinadas. Al nombre de las librerías (en este caso stdio.h) también se le conocen como header, En particular, las funciones de input and output se incluyen en <stio.h>.
  • La función printf es una de varias funcoines dentro de <stdio.h>.
  • Tipos de variables int,float, char. En C es necesario especificar el típo de variable que se va a utilizar, y dependiendo de ello, se asigna una porción de la memoria para su uso. int corresponde a variables de tipo entero. float corresponde a variables reales (con su respectiva limitante de memoria). char corresponde a variables de tipo caracter, con las cuales es posible definir cadenas (strings).
  • El programa principal int main(). Esto define a la función principal del programa. Dentro de esta función se describen las variables que se van a utilizar, las condicionales lógicas que se deben seguir, y los loops básicos for, while, if, … , etc.

Para crear un programa en C debemos seguir los siguientes pasos

  1. Escribir las instrucciones en un archivo de texto programa.c. Para esto necesitaos un editor de texto.
  2. Compilar el archivo programa.c. En Linux el compilador por defecto es GCC. El compilador puede contener muchas instrucciones, es por eso recomendable usar el compilador junto con un archivo Makefile. Este archivo Makefile es un archivo de texto con las opciones que quieren proporcionarse al compilador.
  3. Ejecutar el archivo complilado ./programa.

En los siguientes ejemplos ilustramos precisamente estos pasos. Los programas que describiremos son muy sencillos y el objetivo principal es que tengas un punto de partida para que puedas comenzar a programar en C usando Debian/Linux.

Para conocer más sobre la programación en C te recomendamos ampliamente leer el K&R. En línea es también una gran cantidad de recursos que te van a ayudar a programar en este maravilloso lenguaje.

Práctica

Hola Mundo

  • Desde tu editor de texto favorito crea un archivo de texto que llevará el nombre de hola.c

  • Antes de compilar el archivo, vamos a crear un Makefile con las instrucciones de compilación. Esto te ayudaráen el futuro, cuando trabajes con proyectos más complejos.

  • Compila tu programa (usando Makefile)
make hola

Nota que basta con escribir hola y no es necesario escribir hola.c.

  • Opcional: Compila tu programa (sin usar el Makefile)

Es posible compilar el programa en C sin pasar por el Makefile, para esto es necesario escribir todas las opciones en el proceso 

gcc -pipe -O2 -Wall hola.c -o hola -lm
  • Correr tu programa

Para correr el programa compilado, escribimos desde la terminal lo siguiente: 

./hola

Descripción linea a línea del código:

  • int main(){}

    Ésta es una función especial dentro de C. Es dentro de ésta función que tu programa será ejecutado llamando otras funciones para llevar a cabo las instrucciones deseadas.

    float, int

    En C es necesario declarar el tipo de variables que el programa utilizará, con tal de reservar la porción de memoria correcta. En nuestro caso, int indica variables de tipo entero y float indica variables de tipo flotante (valores con punto decimal).

    Después de declarar el tipo de variable el nombre de la variable puede darse enseguida, en nuestro caso, hemos definido las variables fhar y celsius como variables de tipo flotante, y las variables inferior, superior e incremento como variables de tipo entero.

    Desde luego hay otros tipos de variables que pueden declararse dependiendo de las necesidades. El tamaño y rango en la siguente tabla varía dependiendo del tipo del hardware y el tipo de compilador.

    Identificador Tipo Formato Tamaño en bits Intervalo
    char Carácter %c 8 0 a 255
    int Entero con signo %d 16 -32 768 a 32 767
    float Flotante simple(aprox. 6 dígitos de precisión) %f 32 3.4E-38 a 3.4E+38
    double Flotante doble %lf 64 1.7E-308 a 1.7E+308
    void vacío   0 sin valor
  • inferior = 0

    Esta parte se conoce como la inicialización de las variables. Aquí hemos asignado un valor inicial a cada una de las variables declaradas.

  • printf("Tabla de equivalencia F, C\n")

    La función printf está definida dentro de la librería stdio.h. En este caso, la instrucción es que dentro de la salida estándard (la terminal) se muestren el mensaje "Tabla de equivalencia F, C". Al final "\n" es una instrucción para indicar a printf que debe continuar en una nueva línea. En la siguente tabla se encuentran algunas instrucciones similaries.

    Código Significado
    '\n' nueva línea
    '\r' retorno de carro
    '\f' nueva página (form feed)
    '\t' tabulador horizontal
    '\b' retroceso (backspace)
    '\'' comilla simple
    '\"' comillas
    '\\ ' barra invertida
    '\? ' interrogación
  • while(fahr <= superior){}

    Este es unejemplo de un bucle en C y tiene la siguente estructura:

    while (expression){ sentence }

    En nuestro caso, mientras se verifica que el valor de la variable fahr sea menor o igual al valor de la variable superior, entonces el compilaor procede a realizar las instrucciones especificatas dentro de las llaves {}.

    Existen además otros tipos de bucles en C:

    • la sentencia condicional if.
    • el bucle while.
    • el bucle do - while
    • el bucle for
    • las sentencias condicionale break, continue.
    • la selección de casos switch.

    Algunas senteincias de comparación son

    < menor que
    > mayor que
    <= menor o igual que
    >= mayor o igual que
    == igual que
    != distinto de

    También es posible usar los opeardores lógicos OR: ||, AND: && and NOT: !.

  • celsius = (5.0/9.0) * (fahr - 32.0);

    En esta línea, estamos usando operadores aritméticos para convertir grados F en grados C. A continuación en la tabla se muestran más de estos opeardores:

    + operador de suma
    - operador de resta
    '*' multiplicación
    '/' división entera
    '%' residuo de la división (operador módulo)
  • printf("%4.0f\t%6.2f\n", fahr, celsius);

    Nuevamente nos encontramos con la función printf, pero en esta ocasión usamos más argumentos. printf acepta un especificador de formato, el cual toma la forma %[flags][width][.prec][h|lL] type. Como la variable fahr es de tipo flotante, debemos ingresar en printf el tipo de variable a imprimir en la forma %type, que en este caso sería %f para el tipo flotante. Sin embargo, al escrbibir %4.0f, estamos indicando que queremos imprimir el valor de una variable de tipo flotante con longitud de 4 espacios y 0 espacios para decimales. Al escriibir \t indicamos que queremos un espacio de tipo tabular. %6.2f indica que queremos imprimer el valor de una variable de tipo flotante con longitud de 6 espacios, y 2 espacios después del punto decimal. Finalmente \n indica una nueva linea. Los valores de %f serán tomados de las variables fahr y celsius, en ese orden.

  • fahr = fahr + incremento

    Aquí lo que hacemos es reemplazar el valor de la variable fahr al sumar el valor de incremento. Antes de continuar, el compilador regresará a la expresión a verificar dentro del bucle while. Si el valor de fhar <= superior es verdadero, se ejecutarán nuevamente las instrucciones, de lo contrario, el compilador saldrá del bucle

  • return 0;

    En C y en muchos otros programas, return 0; indica que el programa se ha ejecutado de manera correcta.

Conclusiones

C es un gran lenguaje de programación para iniciar, pero paradójicamente, es un lenguaje muy difícil de usar en un nivel intermedio. Sin embargo, C es la introducción a otros lenguajes y paradigmas de programación de más alto nivel.

Gráficos en R

  • Gráfico de tipo Box Plot.

Este es un gráfico muy utilizado que resume las estadisticas descriptivas de una variable: 

boxplot(df$Value, main="Título del gráfico", col="cyan")

  • Gráfico de funciones matemáticas

Supongamos que queremos hacer la gráfica de la funcion

\begin{equation} f(x) = 3\sin\left(\frac{x}{3}\right). \end{equation}

Para esto, definiremos una sucesión de puntos x con valores desde \(x = -3\) hasta \(x = 3\). 

x <- seq(-3,3,by=0.01)
plot(x, 3*sin(x/3), main="Gráfica de una función", type ="l", col = "red")

  • Para guardar el gráfico como una imagen debmos usar las funciones png() y dev.off()
x <- seq(-3,3,by=0.01)
png("grafico1.png")
plot(x, 3*sin(x/3), main="Gráfica de una función", type ="l", col = "red")
dev.off()

Mi primer programa en BASH

Objetivo

En esta sección, aprenderemos a programar en BASH, usando diversas herramientas dispoinibles en nuestro sistema operativo.

Antecedentes

Bash es muy probabliemente el programa que tu sistema operativo Linux llama la Termial. Es una interfaz en donde el usurario escribe línear de comando las cuales son interpretadas y ejecutadas por una o varios programas dentro del sistema operativo.

Práctica

En esta práctiva, vamos a generar un programa que utiliza diversas herramientas dentro de nuestro sistema operativo.

Suponemos que el usuario tiene instalado en sus systema operativo alguna versión funcional de R y de Libre office.

también vamos a necesitar el programa temperatura que se programó anteriormente, dando la tabla de quivalencias de temperaturas. 

#!/bin/bash
echo "Este es nuesto primer programa escrito en  BASH,"
echo "al final se genera una imagen png con una gráfica."
./temperatura | awk  ' NR>2 {print last} {last=$1","$2}'  > mytemp.csv

Rscript -e '
df  <- read.csv("mytemp.csv",header=FALSE)
df

png("grafica_01.png")
plot(df,type="l")
dev.off()
'

El programa anterior ilustra muy bien, cómo mediante un script en BASH, hemos utilizado

  1. Un programa escrito en C.
  2. Un programa en bash llamado awk para procesar una salida,
  3. Un script escrito en el lenguaje R.

Las posibilidades ahora se muestran enormes!
